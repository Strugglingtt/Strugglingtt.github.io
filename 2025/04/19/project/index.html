流量卡销售引流小程序架构设计方案

一、整体架构设计

[ 用户端小程序 (uni-app) ] <——API——> [ 后端服务 (Gin) ] <——> [ 数据库 (MySQL/GORM) ]
|
+——> [ 文件/图片存储 (本地或云存储) ]

二、功能模块划分

1. 小程序端（uni-app）

  • 首页
    • 搜索框：根据号码搜索流量卡
    • 轮播图：展示三张运营/活动图
    • 卡片商品滑动列表：展示卡片信息，可点击进入详情/下单
  • 菜单栏
    • 首页
    • 客服（可跳转到在线客服或外部客服工具）
    • 我的（需登录才能查看订单、信息）
  • 我的页面
    • 登录/注册（微信授权或手机号验证码）
    • 用户头像、用户名展示
    • 地址管理
    • 订单管理（待付款、待发货、待收货、已完成）

2. 后端服务（Gin + GORM）

  • 用户模块
    • 微信授权登录/手机号登录
    • 用户信息管理
  • 卡片商品模块
    • 卡片信息CRUD
    • 卡片搜索
    • 卡片列表与详情
  • 订单模块
    • 下单、支付（对接微信支付/模拟支付）
    • 订单状态流转（待付款、待发货、待收货、已完成）
    • 订单查询/详情
  • 收货地址模块
    • 地址增删改查（CRUD）
  • 轮播图/活动图模块
    • 图片管理、展示接口
  • 客服模块
    • 客服信息（可配置外链或小程序内部客服）

3. 数据库表设计建议

  • 用户表（user）
  • 卡片商品表（card）
  • 订单表（order）
  • 订单详情表（order_detail）
  • 收货地址表（address）
  • 轮播图表（banner）
  • 客服信息表（customer_service）

三、核心业务流程

  1. 用户进入首页
    • 调用接口拉取轮播图、流量卡商品列表
  2. 搜索功能
    • 前端输入号码关键词，后端提供模糊搜索API
  3. 下单/购买流程
    • 用户点击卡片，进入详情页，点击下单，根据登录状态引导登录
    • 填写/选择收货地址，提交订单
    • 跳转支付页面（微信支付等）
    • 支付完成，订单状态流转
  4. 订单管理
    • 仅登录用户可查看订单列表、订单详情
    • 订单支持状态筛选（待付款、待发货、待收货、已完成）
  5. 我的页面
    • 显示用户头像、用户名
    • 地址列表管理
    • 订单管理快捷入口
  6. 客服入口
    • 客服联系方式、在线客服、或跳转外部小程序/链接

四、权限与安全性

  • 登录态校验（JWT/Session，小程序端持有token）
  • 未登录访问“我的”或订单页时，提示登录
  • 订单等重要操作需校验用户身份
  • API接口需做鉴权与防刷限制

五、关键技术点建议

  • uni-app建议使用vue3+composition-api（如支持），页面用tabBar实现四个菜单
  • 图片（卡片图片、轮播图等）建议存储在云存储（如阿里云OSS、腾讯云COS）或本地静态目录，后端返回图片链接
  • 后端接口遵循RESTful风格，前后端通过Token鉴权
  • GORM用于数据层的ORM操作，数据库建议MySQL
  • 微信小程序端支付需走微信官方流程，后端对接微信支付API
  • 支持分页、搜索、筛选等常用功能
  • 日志与异常处理，接口返回规范化

六、后端接口设计

流量卡小程序后端接口与中间件设计（Golang + Gin + GORM）

一、接口分组总览

  • /api/v1/user 用户相关
  • /api/v1/card 卡片商品相关
  • /api/v1/order 订单相关
  • /api/v1/address 收货地址相关
  • /api/v1/banner 轮播图相关
  • /api/v1/service 客服相关
  • /api/v1/upload 图片上传相关

二、接口详细设计

1. 用户模块（/api/v1/user）

方法 路径 描述 权限
POST /login/wx 微信授权登录 无
POST /login/phone 手机号验证码登录 无
GET /info 获取用户信息 需登录
POST /logout 用户登出 需登录
PUT /update 修改用户信息 需登录

2. 卡片商品模块（/api/v1/card）

方法 路径 描述 权限
GET /list 获取卡片列表 无
GET /search 根据号码搜索卡片 无
GET /:id 获取卡片详情 无
POST /create 新增卡片（后台用） 管理员
PUT /:id 更新卡片（后台用） 管理员
DELETE /:id 删除卡片（后台用） 管理员

示例参数说明：

  • /list 支持分页：?page=1&page_size=10
  • /search 支持号码模糊搜索：?keyword=xxx

3. 订单模块（/api/v1/order）

方法 路径 描述 权限
POST /create 创建订单 需登录
POST /pay/:id 订单支付（微信等） 需登录
GET /list 获取订单列表 需登录
GET /:id 获取订单详情 需登录
PUT /cancel/:id 取消订单 需登录
PUT /confirm/:id 确认收货 需登录

参数说明：

  • /list 支持状态筛选：?status=1&page=1&page_size=10
    状态：1=待付款，2=待发货，3=待收货，4=已完成

4. 收货地址模块（/api/v1/address）

方法 路径 描述 权限
GET /list
